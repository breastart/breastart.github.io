# breastart.github.io

Сайт BREAST ART (Діана Чмих). Статичний сайт для GitHub Pages.

## Як опублікувати
1. Створіть репозиторій з назвою **breastart.github.io** (Public).
2. Завантажте сюди весь вміст цієї папки (Add file → Upload files), включно з папкою `images/`.
3. Settings → Pages → Source: **Deploy from a branch** → `main` / `root` → Save.
4. За 1–2 хвилини сайт буде доступний за адресою **https://breastart.github.io/**

## Як редагувати каталог
Усе — у файлі **data/catalog.json**. Один обʼєкт = одна робота:
- `"status": "available"` або `"sold"` — наявність (продано показується з позначкою);
- `"price"` — ціна в грн (число) або `null`;
- `"image"` — шлях до файлу у `images/`.
Щоб замінити фото — завантажте новий файл з **тим самим іменем** у `images/`.

## Форма замовлення
- Кнопка «Замовити» у каталозі відкриває форму (імʼя, телефон, місто, відділення НП).
- Замовлення надсилається на пошту `ORDER_EMAIL` (зараз `sisidiart@gmail.com`) — за замовчуванням через поштовий застосунок покупця (mailto).
- Щоб замовлення приходили **автоматично** (без поштового застосунку покупця): зареєструйте безкоштовну форму на **formspree.io** з доставкою на цю пошту і вставте ендпоінт у `assets/js/catalog.js` (`FORMSPREE_ENDPOINT`).
- Оплату НЕ беремо на сайті: після підтвердження наявності й термінів ви самі надсилаєте покупцю посилання на оплату.

## Якщо завантаження падає («file too large»)
GitHub-овський веб-аплоадер не любить багато файлів за раз. Картинки вже стиснені під веб (до ~1600px). Якщо все одно падає — завантажуйте папку `images/` окремо і **частинами** (по 30–40 файлів), решту сайту — окремим коммітом.

## Важливо
- Репозиторій публічний: не додавайте паролі, ключі оплати, рукопис книги.
- На сайт ідуть лише web-версії фото; великі оригінали для друку тримайте на Google Drive.
